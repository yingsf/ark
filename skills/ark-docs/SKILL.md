# /ark-docs

## 目标
编写或更新项目文档，使其与真实实现和当前 Artifact 保持一致。

## 前置建议
- 如果 Artifact 文档可能已过期，建议先执行 `/ark-sync` 同步状态

## 适用场景
- README 缺失或过期
- 需要补充功能使用文档
- 安装、运行、开发说明需要更新
- 实现变化影响用户或开发者使用方式

## 不适用场景
- 当前没有任何文档影响
- 请求纯粹是代码实现且尚无文档必要

## 输入
- 当前代码、当前项目结构、现有文档、相关 Artifact

## 输出
- 更新后的 README 或其他说明文档
- 文档与实现的一致性说明
- 尚未文档化部分的说明

## 相关 Artifact
- 可读取所有 docs Artifact
- 可更新仓库中的文档文件

## 工作流
1. 检查真实代码和项目命令。
2. 更新文档以反映当前现实。
3. 保持说明具体、可执行。
4. 删除过期或误导内容。
5. 标记仍需补充的文档空白。

## 验证要求
- 文档内容应与实现一致
- 命令、路径、文件名应合理且一致

## 停止条件
- 文档已足以支持当前使用与开发

## 备注
不要为尚未实现的功能生成看起来很完整的虚假文档。
